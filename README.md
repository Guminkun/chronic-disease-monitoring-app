# HealthMonitor - 慢性病智能监测平台

## 项目概述

HealthMonitor是一款基于现代web技术的跨端慢性病监测应用，为患者和医生提供完整的健康数据管理和医患沟通解决方案。该项目旨在帮助慢性病患者进行自我健康监测，同时支持医生查看患者数据进行远程医疗指导。

### 核心功能

- **患者端**：检查报告录入、健康指标趋势分析、慢性病管理、医患沟通
- **医生端**：患者列表查看、健康数据监测、临床备注、实时消息沟通
- **医患绑定**：灵活的医患关系管理和隐私保护
- **数据可视化**：多种图表展示健康指标趋势

## 重要说明

⚠️ **APP端和WEB端是独立的系统**

- **APP端** (`/app`): 患者和医生的移动应用，使用手机号登录
- **WEB端** (`/app/web`): 管理员管理后台，独立的登录系统和UI设计
- 两个端口都支持手机号+默认密码(123456)登录
- 后续任何功能调整都需要在README中同步更新

1. **心血管疾病** - 高血压、冠心病、心力衰竭、心律失常等
2. **代谢/内分泌** - 糖尿病、甲状腺疾病、痛风、高脂血症等
3. **呼吸系统** - COPD、哮喘、慢性支气管炎、肺纤维化
4. **消化系统** - 慢性胃炎、消化性溃疡、肝硬化、慢性肠炎
5. **泌尿系统** - 慢性肾炎、肾病综合征、慢性肾衰竭
6. **神经系统** - 帕金森病、阿尔茨海默病、癫痫、多发性硬化
7. **血液系统** - 贫血、再生障碍性贫血、白血病、淋巴瘤
8. **风湿免疫** - 类风湿关节炎、红斑狼疮、强直性脊柱炎
9. **皮肤疾病** - 银屑病、湿疹、白癜风
10. **精神类** - 抑郁症、焦虑症、精神分裂症、双相情感障碍
11. **其他** - 艾滋病、慢性疼痛、骨质疏松等

## WEB管理端设计原则

### 四大核心设计原则

1. **轻量化适配**
   - 模块功能极简聚焦，不设计复杂工作流、多级审批
   - 所有接口设计为基础CRUD和简单统计查询
   - 完美适配Express/FastAPI/Gin等轻量级后端框架
   - 无需复杂中间件，降低接口开发与后台渲染成本

2. **业务闭环**
   - 完全匹配APP端核心功能的后台管控
   - 四大核心业务全覆盖：用户、病种指标、报告、医患绑定
   - 从用户注册审核→病种指标配置→报告数据录入→医患数据共享，全流程在后台可管控
   - 不存在功能缺口，前后端完整对应

3. **安全合规**
   - 医疗健康数据属于敏感信息，必须内置权限控制、操作审计、数据合规
   - 满足《个人信息保护法》要求
   - 细粒度权限配置：避免越权操作
   - 完整的操作日志审计：记录所有增删改查操作
   - 数据脱敏规则：保护敏感个人信息（如手机号隐藏中间四位）

4. **易运维**
   - 适配`192.168.88.100:5432`的PostgreSQL数据库
   - 提供基础监控与配置：数据库连接监控、连接池管理
   - 数据备份提醒、历史数据归档策略
   - 无需复杂中间件，简化运维负担

## 模块架构详解

### 核心必选模块（上线必须）

项目上线前必须具备这5个模块，它们直接支撑APP正常运行，是后台的基础，无冗余设计，开发成本低。

#### 1. 用户与权限管理模块

**1.1 账号管理**
- 支持按账号、手机号、姓名、角色（患者/医生）、注册时间筛选查询所有用户
- 基础操作：查看用户详情、禁用/启用违规账号、重置登录密码、批量导出用户数据
- 区分患者账号、医生账号的展示标签，医生账号单独标注审核状态

**1.2 医生资质审核**
- 专门的审核列表：待审核、已通过、已驳回
- 查看医生上传的执业医师证号、所属医院、科室等资质信息
- 通过/驳回操作，驳回时可填写原因同步至APP端

**1.3 后台角色与权限管理**
- 预设3个角色：超级管理员、普通管理员、技术管理员
- 细粒度权限配置：控制不同角色可查看/操作的模块
- 模块级权限隔离：避免越权操作

#### 2. 慢性病与检查指标管理模块

支撑APP端用户选择慢性病、报告录入、趋势图生成的核心业务，实现数据字典统一管控。

**2.1 慢性病字典管理**
- 维护60+慢性病列表，支持按疾病分类、疾病名称筛选
- 操作：新增、编辑、删除、启用/禁用病种
- 控制APP端可选择的病种范围

**2.2 检查项目与指标管理**
- 维护所有检查类型：血常规、肝功能、肾功能、血糖、血脂等
- 维护每个检查类型下的指标名称、参考范围、单位、正常/异常判定标准
- 为OCR识别、趋势图、异常预警提供标准依据

**2.3 病种-指标关联配置**
- 将慢性病与对应检查指标做绑定
- 例如：糖尿病→空腹血糖、餐后2h血糖、糖化血红蛋白
- APP端用户选择病种后自动展示关联指标，无需前端硬编码

#### 3. 检查报告数据管理模块

管理APP端OCR识别上传、手动录入的所有报告数据，是医疗数据的核心管控模块。

**3.1 报告列表管理**
- 多维度筛选：用户ID/姓名、关联慢性病、检查类型、检查日期、录入方式（OCR/手动）
- 基础操作：查看报告完整数据、查看OCR原始图片、编辑错误数据、删除无效报告、批量导出

**3.2 报告附件管理**
- 统一管理用户上传的报告单图片
- 支持查看、批量清理冗余/过期图片，释放存储资源
- 配置图片存储有效期、最大上传大小

#### 4. 医患绑定关系管理模块

管控APP端医患绑定行为，防止违规绑定、越权查看数据、恶意绑定。

**4.1 绑定关系查询**
- 按医生ID/姓名、患者ID/姓名、绑定时间、绑定状态筛选
- 查看绑定渠道（扫码绑定/验证码绑定）、绑定码信息

**4.2 绑定关系管控**
- 支持管理员主动解除异常绑定关系
- 场景：医生离职、患者投诉违规查看
- 同步更新数据库`t_doctor_patient`表的状态

**4.3 绑定日志审计**
- 记录所有绑定、解绑、管理员解除操作日志
- 用于问题追溯

#### 5. 系统基础配置模块

适配轻量化后端与PostgreSQL数据库，完成基础参数配置。

**5.1 数据库配置监控**
- 展示PostgreSQL连接信息（192.168.88.100:5432）
- 监控连接状态、连接池使用情况
- 配置数据库备份提醒、历史数据归档规则

**5.2 APP基础参数配置**
- 医患绑定验证码有效期
- 密码规则（长度、复杂度）
- 图片上传格式/大小限制
- OCR识别超时时间

**5.3 第三方服务配置**
- OCR服务：API Key、Secret、接口地址、调用配额告警
- 二维码服务：生成参数、过期策略

### 可选扩展模块（后期迭代）

初期项目上线可暂不开发，用户量增长、业务复杂化后再迭代。

#### 1. 数据统计与可视化模块

给管理员提供数据运营能力，无需接入重型BI工具。

- **用户运营统计**：患者/医生注册量、日活/月活、地域分布、病种分布
- **报告数据统计**：报告录入总量、OCR识别成功率、各指标异常率
- **医患关系统计**：医生绑定患者数量、平均绑定数、医生活跃度分析

#### 2. 异常数据监控与预警模块

- **指标异常告警**：基于预设参考范围，自动标记超阈值报告，生成异常列表
- **违规行为监控**：监控医生频繁查看患者数据、批量绑定等异常行为，推送告警

#### 3. 合规与审计管理模块

针对医疗数据合规要求，强化数据安全管控。

- **操作日志全审计**：记录后台管理员的所有新增、编辑、删除、导出操作，留存日志用于合规审查
- **用户数据合规管理**：处理用户数据注销、导出申请，审核通过后执行，留存审核记录
- **数据脱敏配置**：配置报表、导出数据的脱敏规则

## 权限控制矩阵

| 后台角色 | 可操作模块 | 不可操作模块 |
|---------|---------|---------|
| **超级管理员** | 所有模块，拥有全部增删改查权限 | 无 |
| **普通管理员** | 用户管理、医生资质审核、慢性病指标管理、报告管理、绑定关系管理、统计模块 | 系统配置、第三方服务配置、数据库监控 |
| **技术管理员** | 系统基础配置、第三方服务配置、数据库监控、操作日志审计 | 用户数据、报告数据的增删改 |

## 模块设计合理性总结

1. **贴合业务流程**：从用户注册审核→病种指标配置→报告数据录入→医患数据共享，全流程在后台可管控，和APP端功能完全对应，不存在功能缺口

2. **适配轻量化技术栈**：所有模块接口均为基础CRUD、简单统计查询，无高并发、分布式事务等重型需求，完美适配Express/FastAPI/Gin，无需引入额外中间件

3. **兼顾安全与成本**：必选模块满足上线刚需，开发周期短；扩展模块按需迭代，避免前期资源浪费。同时通过权限控制、日志审计、数据脱敏，保障敏感数据的安全合规

4. **易于后期迭代**：模块间耦合度低，新增功能时只需新增对应管理模块，不会影响现有核心模块运行
