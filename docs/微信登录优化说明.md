# 微信登录优化说明

## 需求

优化微信小程序登录流程：
1. 首次打开小程序时，询问用户确认登录
2. 用户确认后进行微信登录
3. 之后打开小程序时自动登录，无需再次确认

## 实现方案

### 核心逻辑

```
用户打开小程序
    ↓
检查token是否存在
    ├─ 存在 → 直接进入首页 ✅
    └─ 不存在 → 检查是否登录过
        ├─ 登录过 → 自动登录 ✅
        └─ 未登录过 → 显示登录页面
            └─ 用户点击"微信一键登录"
                ├─ 首次 → 显示确认弹窗
                └─ 确认后 → 登录 ✅
```

### 关键技术点

#### 1. 本地存储标记

**存储项：** `hasLoggedInBefore`

**作用：** 标记用户是否曾经登录过

**值：** `true` / `false`

#### 2. 登录流程优化

**文件：** `frontend-mobile/src/App.vue`

**应用启动时：**
```typescript
onLaunch(() => {
  // 检查token
  if (token存在) {
    恢复用户状态()
  } else if (hasLoggedInBefore) {
    自动登录()
  }
})
```

**文件：** `frontend-mobile/src/pages/login/login.vue`

**登录页面加载时：**
```typescript
onMounted(() => {
  if (token存在) {
    直接跳转首页()
  } else if (hasLoggedInBefore) {
    自动登录()
  }
  // 否则等待用户手动点击登录
})
```

**用户点击"微信一键登录"：**
```typescript
if (!hasLoggedInBefore) {
  显示确认弹窗("是否使用微信账号登录小二？")
  用户确认后 → 登录()
} else {
  直接登录()
}
```

## 修改文件

### 1. App.vue

**添加功能：**
- 应用启动时自动检查登录状态
- 自动恢复用户信息
- 自动尝试登录（如果之前登录过）

**关键代码：**
```typescript
const autoLoginIfNeeded = async () => {
  const token = uni.getStorageSync('token');
  
  if (token) {
    await userStore.fetchCurrentUser();
    return;
  }
  
  const hasLoggedInBefore = uni.getStorageSync('hasLoggedInBefore');
  if (hasLoggedInBefore) {
    await userStore.loginByWechat();
  }
};
```

### 2. login.vue

**修改功能：**
- 优化 `onMounted`：检查登录状态
- 新增 `checkLoginStatus`：判断是否需要自动登录
- 修改 `handleWechatLogin`：首次登录显示确认弹窗
- 新增 `performWechatLogin`：执行登录逻辑

**关键代码：**
```typescript
// 首次登录确认
if (!hasLoggedInBefore) {
  uni.showModal({
    title: '微信登录',
    content: '是否使用微信账号登录小二？',
    confirmText: '确认登录',
    success: async (res) => {
      if (res.confirm) {
        await performWechatLogin();
      }
    }
  });
}
```

## 用户体验流程

### 场景1：首次使用

```
用户首次打开小程序
    ↓
显示登录页面
    ↓
用户点击"微信一键登录"
    ↓
弹出确认框："是否使用微信账号登录小二？"
    ↓
用户点击"确认登录"
    ↓
登录成功，进入首页
    ↓
保存标记：hasLoggedInBefore = true
```

### 场景2：再次使用（token有效）

```
用户打开小程序
    ↓
App.vue检测到token存在
    ↓
自动恢复用户状态
    ↓
直接进入首页 ✅（无需任何操作）
```

### 场景3：再次使用（token过期）

```
用户打开小程序
    ↓
App.vue检测到token不存在
    ↓
检测到 hasLoggedInBefore = true
    ↓
自动尝试微信登录
    ↓
登录成功，进入首页 ✅（无需任何操作）
```

### 场景4：清除缓存后使用

```
用户打开小程序
    ↓
token和hasLoggedInBefore都不存在
    ↓
显示登录页面
    ↓
用户点击"微信一键登录"
    ↓
弹出确认框（因为hasLoggedInBefore不存在）
    ↓
用户确认后登录
```

## 优势对比

| 项目 | 优化前 | 优化后 |
|------|--------|--------|
| 首次登录 | 直接登录，无确认 | 弹窗确认，更安全 ✅ |
| 再次使用 | 需要手动点击登录 | 自动登录，无感知 ✅ |
| Token有效 | 需要手动登录 | 直接进入首页 ✅ |

## 注意事项

1. **数据持久化**
   - `token` 和 `hasLoggedInBefore` 都存储在本地
   - 用户清除缓存后会重新询问确认

2. **安全性**
   - 首次登录需要用户明确确认
   - 避免未经授权的自动登录

3. **兼容性**
   - 仅在微信小程序环境启用
   - 使用条件编译 `#ifdef MP-WEIXIN`

4. **错误处理**
   - 自动登录失败时清除标记
   - 避免反复尝试失败
