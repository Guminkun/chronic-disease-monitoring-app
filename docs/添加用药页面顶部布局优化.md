# 添加用药页面顶部布局优化

## 问题描述

**问题：** 添加用药页面顶部超出屏幕，影响视觉体验。

**原因：** 页面未考虑设备安全区域（刘海屏、状态栏高度），导致内容与系统状态栏重叠。

## 解决方案

### 修改文件
`frontend-mobile/src/pages/medication/add.vue`

### 修改内容

**修改前：**
```scss
.medication-page {
  min-height: 100vh;
  background: #eef2f7;
  display: flex;
  flex-direction: column;
}

.scroll-content {
  flex: 1;
  padding: 24rpx 32rpx;
  box-sizing: border-box;
}
```

**修改后：**
```scss
.medication-page {
  min-height: 100vh;
  background: #eef2f7;
  display: flex;
  flex-direction: column;
  padding-top: constant(safe-area-inset-top);      /* iOS 11.0-11.2 */
  padding-top: env(safe-area-inset-top);            /* iOS 11.2+ */
}

.scroll-content {
  flex: 1;
  padding: 24rpx 32rpx;
  padding-top: calc(24rpx + constant(safe-area-inset-top));
  padding-top: calc(24rpx + env(safe-area-inset-top));
  box-sizing: border-box;
}
```

## 技术说明

### safe-area-inset-top

**作用：** 自动获取设备安全区域顶部高度，适配刘海屏、状态栏等。

**兼容性：**
- `constant(safe-area-inset-top)` - iOS 11.0-11.2
- `env(safe-area-inset-top)` - iOS 11.2+

**设备适配：**
| 设备类型 | safe-area-inset-top |
|---------|---------------------|
| 普通 iPhone | 20px（状态栏） |
| iPhone X/11/12/13 | 44px（刘海屏） |
| iPhone 14 Pro/15 Pro | 59px（灵动岛） |
| Android | 通常为状态栏高度 |

## 注意事项

1. **双重padding**
   - `.medication-page` 和 `.scroll-content` 都添加了安全区域padding
   - 这是必要的，确保整个页面布局正确

2. **calc() 函数**
   - 使用 `calc()` 确保原有 padding (24rpx) 不被覆盖
   - 而是在原有基础上增加安全区域高度

3. **向后兼容**
   - 使用 `constant()` 和 `env()` 确保兼容不同iOS版本
   - 在不支持的设备上，会忽略这些属性
