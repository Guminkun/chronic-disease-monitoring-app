# 首页待办提醒优化说明

## 修改内容

### 问题
首页的待办提醒显示了已处理的消息，影响用户体验。

### 解决方案
修改 `loadLatestNotifications` 函数，添加 `is_handled: false` 参数，只加载未处理的消息。

### 修改代码

**文件：** `frontend-mobile/src/pages/index/index.vue`

**修改前：**
```typescript
const loadLatestNotifications = async () => {
  if (!userStore.token) { latestNotifications.value = []; return }
  try {
    const res = await getNotifications({ all_members: true, limit: 5 })
    latestNotifications.value = res.items || []
  } catch { latestNotifications.value = [] }
}
```

**修改后：**
```typescript
const loadLatestNotifications = async () => {
  if (!userStore.token) { latestNotifications.value = []; return }
  try {
    const res = await getNotifications({ 
      all_members: true, 
      is_handled: false,  // 🔧 只加载未处理的消息
      limit: 5 
    })
    latestNotifications.value = res.items || []
  } catch { latestNotifications.value = [] }
}
```

## 影响范围

### 首页通知展示区域

1. **顶部通知卡片**
   - 显示最近的通知列表
   - 现在只显示未处理的消息
   - 位置：首页顶部展开的通知面板

2. **待办提醒卡片**
   - 显示待办事项列表
   - 现在只显示未处理的消息
   - 位置：首页中间的"待办提醒"卡片

### 用户流程

```
用户打开首页
    ↓
加载最新通知（过滤已处理消息）
    ↓
显示未处理的消息
    ↓
用户点击消息
    ↓
跳转到消息中心或对应模块
    ↓
标记消息为已处理
    ↓
下次刷新首页时，该消息不再显示
```

## 功能对比

| 功能点 | 优化前 | 优化后 |
|--------|--------|--------|
| 通知显示 | 显示所有消息 | 只显示未处理消息 |
| 已处理消息 | 仍显示在首页 | 不在首页显示 |
| 待办提醒 | 包含已处理消息 | 只显示待处理消息 |
| 用户体验 | 可能重复看到已处理消息 | 只看到需要处理的消息 |

## 测试验证

### 测试场景

1. **新用户首页**
   - 没有消息时，不显示通知卡片
   - 有未处理消息时，显示消息列表

2. **有消息的用户**
   - 显示未处理的消息
   - 不显示已处理的消息

3. **处理消息后**
   - 在消息中心标记消息为已处理
   - 返回首页刷新
   - 已处理的消息不再显示

## 注意事项

1. **API兼容性**
   - 后端已支持 `is_handled` 参数
   - 前端需要传递该参数才能过滤

2. **性能优化**
   - 只加载未处理消息，减少数据传输
   - 提升首页加载速度

3. **用户体验**
   - 避免重复展示已处理的消息
   - 聚焦于待处理事项

## 后续优化建议

1. **消息优先级排序**
   - 按优先级和紧急程度排序
   - 重要消息优先展示

2. **消息分组**
   - 按成员分组显示
   - 按类型分组显示

3. **快速操作**
   - 首页直接标记已处理
   - 无需跳转到消息中心

4. **智能提醒**
   - 根据用户行为智能推荐
   - 重要消息主动推送
